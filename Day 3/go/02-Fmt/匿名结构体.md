



## 匿名结构体详解

**匿名结构体**（Anonymous Struct）是指**没有名字的结构体**，直接定义并使用。

### 对比示例

```go
// 👎 具名结构体 - 先定义类型
type Person struct {
    name string
    age  int
}
p := Person{name: "张三", age: 20}
```

```go
// 👍 匿名结构体 - 直接定义并使用
p := struct {
    name string
    age  int
}{name: "张三", age: 20}
```

### 使用场景

| 场景 | 说明 |
|------|------|
| **临时数据** | 只用一次的简单数据结构，不需要单独定义类型 |
| **测试数据** | 单元测试中快速构造测试对象 |
| **Map 的值** | 存储一些关联的临时数据 |
| **函数返回值** | 返回多个相关联的临时字段 |

### 实际例子

```go
// 场景1：临时封装数据
data := struct {
    ID   int
    Name string
}{ID: 1, Name: "测试"}

// 场景2：Map 的 value 类型
users := map[string]struct {
    Email string
    Phone string
}{
    "张三": {Email: "zhangsan@example.com", Phone: "13800138000"},
}
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| 代码简洁，无需额外定义类型 | 无法复用，每次都要重新定义结构 |
| 适合临时场景 | 代码可读性稍差（没有类型名称） |
| 灵活，快速 prototyping | 不能作为函数参数类型（需要重复定义） |

---

**总结**：匿名结构体就像"一次性"的容器，用完就丢；具名结构体则是" reusable "的模板，可以重复使用。根据场景选择即可！
