# 数据库相关知识梳理

本文系统梳理数据库核心知识，涵盖 SQL 基础（增删改查、联表查询、索引）和 MySQL 常用操作（事务、分页、条件查询），帮助快速建立知识框架。

---

## 一、SQL 基础

### 1. 数据库和表的操作

#### 1.1 数据库操作
- **创建数据库**：
  ```sql
  CREATE DATABASE database_name;
  CREATE DATABASE IF NOT EXISTS database_name CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  ```
- **查看数据库**：
  ```sql
  SHOW DATABASES;
  SHOW CREATE DATABASE database_name;  -- 查看数据库创建语句
  ```
- **选择数据库**：
  ```sql
  USE database_name;
  ```
- **修改数据库**：
  ```sql
  ALTER DATABASE database_name CHARACTER SET utf8mb4;
  ```
- **删除数据库**：
  ```sql
  DROP DATABASE database_name;
  DROP DATABASE IF EXISTS database_name;  -- 安全删除
  ```

#### 1.2 表操作
- **创建表**：
  ```sql
  CREATE TABLE table_name (
      id INT PRIMARY KEY AUTO_INCREMENT,
      name VARCHAR(50) NOT NULL,
      age INT DEFAULT 0,
      email VARCHAR(100) UNIQUE,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
  );
  ```
- **查看表**：
  ```sql
  SHOW TABLES;                    -- 查看当前数据库所有表
  SHOW CREATE TABLE table_name;   -- 查看表创建语句
  DESC table_name;                -- 查看表结构
  SHOW COLUMNS FROM table_name;   -- 查看表字段信息
  ```
- **修改表结构**：
  ```sql
  -- 添加列
  ALTER TABLE table_name ADD COLUMN new_column VARCHAR(50);
  ALTER TABLE table_name ADD COLUMN new_column INT AFTER existing_column;  -- 指定位置
  
  -- 修改列
  ALTER TABLE table_name MODIFY COLUMN column_name VARCHAR(100);  -- 修改数据类型
  ALTER TABLE table_name CHANGE old_name new_name VARCHAR(50);    -- 重命名并修改类型
  
  -- 删除列
  ALTER TABLE table_name DROP COLUMN column_name;
  
  -- 重命名表
  ALTER TABLE old_table_name RENAME TO new_table_name;
  RENAME TABLE old_name TO new_name;
  
  -- 修改表引擎
  ALTER TABLE table_name ENGINE = InnoDB;
  
  -- 添加/删除主键
  ALTER TABLE table_name ADD PRIMARY KEY (column_name);
  ALTER TABLE table_name DROP PRIMARY KEY;
  
  -- 添加/删除外键
  ALTER TABLE table_name ADD CONSTRAINT fk_name FOREIGN KEY (column_name) REFERENCES other_table(id);
  ALTER TABLE table_name DROP FOREIGN KEY fk_name;
  ```
- **删除表**：
  ```sql
  DROP TABLE table_name;
  DROP TABLE IF EXISTS table_name;  -- 安全删除
  TRUNCATE TABLE table_name;        -- 清空表数据，保留表结构（比 DELETE 快）
  ```
- **复制表**：
  ```sql
  CREATE TABLE new_table LIKE old_table;              -- 只复制表结构
  CREATE TABLE new_table AS SELECT * FROM old_table;  -- 复制表结构和数据
  ```

### 2. 数据的增删改查（CRUD）
- **INSERT**：向表中插入新记录
  ```sql
  INSERT INTO table_name (column1, column2) VALUES (value1, value2);
  ```
- **SELECT**：从表中查询数据
  ```sql
  SELECT column1, column2 FROM table_name WHERE condition;
  ```
- **UPDATE**：修改表中现有记录
  ```sql
  UPDATE table_name SET column1 = value1 WHERE condition;
  ```
- **DELETE**：删除表中记录
  ```sql
  DELETE FROM table_name WHERE condition;
  ```
*注意：更新和删除操作务必指定 WHERE 条件，避免误操作全表数据。*

### 3. 联表查询（JOIN）
联表查询用于从多个表中联合检索数据，基于表之间的关联字段。

- **INNER JOIN**：返回两个表中匹配的行（交集）
  ```sql
  SELECT * FROM table1 INNER JOIN table2 ON table1.id = table2.id;
  ```
- **LEFT JOIN**：返回左表全部行，右表匹配的行，无匹配则显示 NULL
  ```sql
  SELECT * FROM table1 LEFT JOIN table2 ON table1.id = table2.id;
  ```
- **RIGHT JOIN**：返回右表全部行，左表匹配的行，无匹配则显示 NULL
  ```sql
  SELECT * FROM table1 RIGHT JOIN table2 ON table1.id = table2.id;
  ```
- **FULL JOIN**：返回左右表所有行（MySQL 不直接支持，可通过 UNION 模拟）
  ```sql
  SELECT * FROM table1 LEFT JOIN table2 ON ... UNION SELECT * FROM table1 RIGHT JOIN table2 ON ...;
  ```
- **自连接**：将同一张表视为两个不同表进行连接，常用于层级结构查询
- **联合查询（UNION）**：合并多个 SELECT 结果集，自动去重；UNION ALL 保留重复

### 4. 索引
索引是数据库提高查询效率的数据结构（类似书的目录），但会占用额外存储并影响写性能。

- **概念**：对表中一列或多列的值进行排序，加快数据检索速度。
- **常见类型**：
    - **主键索引**：唯一标识一行，自动创建，不可为 NULL
    - **唯一索引**：索引列值必须唯一，可为 NULL
    - **普通索引**：无唯一性限制，仅加速查询
    - **全文索引**：用于文本内容的快速搜索（MyISAM 和 InnoDB 支持）
    - **组合索引**：基于多列创建的索引，遵循最左前缀原则

- **索引操作命令**：
  
  **创建索引**：
  ```sql
  -- 创建普通索引
  CREATE INDEX idx_name ON table_name (column_name);
  CREATE INDEX idx_multi ON table_name (column1, column2);  -- 组合索引
  
  -- 创建唯一索引
  CREATE UNIQUE INDEX idx_unique ON table_name (column_name);
  
  -- 创建全文索引
  CREATE FULLTEXT INDEX idx_fulltext ON table_name (column_name);
  
  -- 使用 ALTER TABLE 创建索引
  ALTER TABLE table_name ADD INDEX idx_name (column_name);
  ALTER TABLE table_name ADD UNIQUE idx_unique (column_name);
  ALTER TABLE table_name ADD FULLTEXT idx_fulltext (column_name);
  
  -- 创建表时直接定义索引
  CREATE TABLE users (
      id INT PRIMARY KEY,
      username VARCHAR(50),
      email VARCHAR(100),
      INDEX idx_username (username),           -- 普通索引
      UNIQUE KEY idx_email (email),            -- 唯一索引
      FULLTEXT KEY idx_bio (bio)               -- 全文索引
  );
  ```
  
  **索引相关关键字说明**：
  - **INDEX**：创建普通索引，可以有重复值
  - **UNIQUE**：创建唯一索引，索引列的值必须唯一（NULL 除外，可以有多个 NULL）
  - **PRIMARY KEY**：主键索引，唯一且不能为 NULL，一个表只能有一个主键
  - **FULLTEXT**：全文索引，用于文本搜索，支持 MATCH AGAINST 语法
  - **KEY**：与 INDEX 同义，可互换使用
  - **SPATIAL**：空间索引，用于地理空间数据类型（GEOMETRY、POINT 等）
  
  **使用示例**：
  ```sql
  -- 在创建表时使用各种索引关键字
  CREATE TABLE products (
      id INT AUTO_INCREMENT,
      product_code VARCHAR(50) NOT NULL,
      name VARCHAR(100) NOT NULL,
      description TEXT,
      price DECIMAL(10,2),
      stock INT DEFAULT 0,
      
      PRIMARY KEY (id),                        -- 主键索引
      UNIQUE INDEX uk_code (product_code),     -- 唯一索引（使用 INDEX）
      UNIQUE KEY uk_name (name),               -- 唯一索引（使用 KEY）
      INDEX idx_price (price),                 -- 普通索引
      FULLTEXT INDEX ft_desc (description)     -- 全文索引
  );
  
  -- 使用 ALTER TABLE 添加不同类型的索引
  ALTER TABLE products ADD INDEX idx_stock (stock);
  ALTER TABLE products ADD UNIQUE uk_sku (sku);
  ALTER TABLE products ADD FULLTEXT ft_name (name);
  ```
  
  **查看索引**：
  ```sql
  SHOW INDEX FROM table_name;
  SHOW KEYS FROM table_name;
  SHOW CREATE TABLE table_name;  -- 查看表结构和索引
  ```
  
  **删除索引**：
  ```sql
  DROP INDEX index_name ON table_name;
  ALTER TABLE table_name DROP INDEX index_name;
  
  -- 删除主键索引
  ALTER TABLE table_name DROP PRIMARY KEY;
  ```
  
  **重命名索引**（MySQL 5.7+）：
  ```sql
  ALTER TABLE table_name RENAME INDEX old_index_name TO new_index_name;
  ```

- **使用原则**：
    - 为经常用于 WHERE、JOIN、ORDER BY 的列创建索引
    - 避免在小数据量表或不常查询的列上建索引
    - 组合索引注意列的顺序，将最常用、选择性最高的列放在最左
    - 索引并非越多越好，需权衡查询与写操作的性能
    - 避免在频繁更新的列上创建过多索引

## 二、MySQL 常用操作

### 1. 事务
事务是一组要么全部成功、要么全部失败的数据库操作，保证数据一致性。

- **ACID 特性**：
    - **原子性（Atomicity）**：事务不可分割，全部执行或全部不执行
    - **一致性（Consistency）**：事务前后，数据库完整性约束未被破坏
    - **隔离性（Isolation）**：并发事务互不干扰
    - **持久性（Durability）**：事务提交后，数据永久保存
- **事务控制语句**：
  ```sql
  START TRANSACTION;   -- 或 BEGIN
  -- 执行 SQL 操作
  COMMIT;              -- 提交事务
  ROLLBACK;            -- 回滚事务
  SAVEPOINT sp_name;   -- 设置保存点，可部分回滚
  ```
- **隔离级别**（由低到高）：
    - **读未提交（READ UNCOMMITTED）**：可能读到未提交数据（脏读）
    - **读已提交（READ COMMITTED）**：只能读到已提交数据（Oracle 默认）
    - **可重复读（REPEATABLE READ）**：同一事务多次读取结果一致（MySQL InnoDB 默认）
    - **串行化（SERIALIZABLE）**：事务串行执行，性能最低
      *查看/设置隔离级别：*
  ```sql
  SELECT @@transaction_isolation;
  SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;
  ```

### 2. 分页
当查询结果集过大时，通过分页返回部分数据，提升用户体验和系统性能。

- **基本语法**：使用 `LIMIT` 和 `OFFSET`
  ```sql
  SELECT * FROM table_name ORDER BY id LIMIT page_size OFFSET offset;
  -- 或简写：LIMIT offset, page_size
  ```
  例如：每页 10 条，查询第 3 页（记录 21~30）
  ```sql
  SELECT * FROM users ORDER BY id LIMIT 10 OFFSET 20;   -- OFFSET = (页码-1)*页大小
  ```
- **优化建议**：
    - 确保 ORDER BY 的字段有索引，避免大偏移量时全表扫描
    - 对于大偏移量，可使用子查询先定位起始 ID：
      ```sql
      SELECT * FROM users WHERE id > (SELECT id FROM users ORDER BY id LIMIT 1 OFFSET 10000) LIMIT 10;
      ```

### 3. 条件查询
条件查询通过 `WHERE` 子句筛选数据，是 SELECT/UPDATE/DELETE 的核心部分。

- **比较运算符**：`=`, `!=`, `>`, `<`, `>=`, `<=`
- **逻辑运算符**：`AND`, `OR`, `NOT`
- **模糊查询**：`LIKE`（`%` 匹配任意多个字符，`_` 匹配单个字符）
- **范围查询**：
    - `BETWEEN ... AND ...` 包含边界值
    - `IN (...)` 匹配列表中的任意值
- **空值判断**：`IS NULL` 或 `IS NOT NULL`（注意不能用 `= NULL`）
- **子查询**：将一个查询的结果作为另一个查询的条件
  ```sql
  SELECT * FROM table1 WHERE column1 IN (SELECT column1 FROM table2 WHERE condition);
  ```
- **其他常用**：`DISTINCT` 去重，`ORDER BY` 排序，`GROUP BY` 分组，`HAVING` 对分组结果过滤

## 总结
掌握 SQL 基础和 MySQL 常用操作是数据库应用的关键。**增删改查**是数据操作的核心，**联表查询**实现多表关联，**索引**优化查询效率；**事务**保证数据一致性，**分页**管理大数据量返回，**条件查询**灵活筛选数据。实际开发中需根据场景合理运用，并注意性能优化与安全性（如防止 SQL 注入）。
