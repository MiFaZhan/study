# 线性查找（Linear Search）

## 概念
线性查找是最简单的查找算法，从数组的第一个元素开始，逐个比较，直到找到目标元素或遍历完整个数组。

## 特点
- **简单直观**：实现非常简单，容易理解
- **无需排序**：不要求数组有序
- **适用范围广**：适用于任何数据类型
- **效率较低**：对于大数据集效率不高

## 时间复杂度
- **最好情况**：O(1) - 第一个元素就是目标
- **最坏情况**：O(n) - 目标在最后或不存在
- **平均情况**：O(n) - 需要遍历约一半的元素

## 空间复杂度
- O(1) - 只需要常数级别的额外空间

## 适用场景
1. 数据量较小
2. 数组无序
3. 只需要查找一次
4. 实现简单性比效率更重要

## 优缺点

### 优点
- 实现简单
- 不需要数据有序
- 适用于链表等非连续存储结构

### 缺点
- 效率低，时间复杂度为 O(n)
- 不适合大数据集

## 与二分查找的对比
| 特性 | 线性查找 | 二分查找 |
|------|---------|---------|
| 时间复杂度 | O(n) | O(log n) |
| 数据要求 | 无序即可 | 必须有序 |
| 实现难度 | 简单 | 中等 |
| 适用场景 | 小数据集 | 大数据集 |
