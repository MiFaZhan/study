# 双向链表与循环链表详解

## 一、双向链表（Doubly Linked List）

### 1.1 概念

双向链表的每个节点包含三个部分：
- **数据域**：存储数据
- **前驱指针（Prev）**：指向前一个节点
- **后继指针（Next）**：指向后一个节点

```
    ┌─────┬─────┬─────┐     ┌─────┬─────┬─────┐     ┌─────┬─────┬─────┐
    │Prev │ Val │Next │────>│Prev │ Val │Next │────>│Prev │ Val │Next │
    │ nil │  1  │  ●  │     │  ●  │  2  │  ●  │     │  ●  │  3  │ nil │
    └─────┴─────┴─────┘     └─────┴─────┴─────┘     └─────┴─────┴─────┘
       ↑                       ↑  │                     ↑  │
       └───────────────────────┘  └─────────────────────┘
```

### 1.2 节点结构定义

```go
type DoublyNode[T any] struct {
    Val  T
    Prev *DoublyNode[T]  // 前驱指针
    Next *DoublyNode[T]  // 后继指针
}
```

### 1.3 基本操作

#### 创建双向链表

```go
func createDoublyList[T any](values []T) *DoublyNode[T] {
    if len(values) == 0 {
        return nil
    }
    
    head := &DoublyNode[T]{Val: values[0]}
    current := head
    
    for i := 1; i < len(values); i++ {
        newNode := &DoublyNode[T]{
            Val:  values[i],
            Prev: current,  // 设置前驱指针
        }
        current.Next = newNode
        current = newNode
    }
    
    return head
}
```

#### 在头部插入节点

```go
func insertAtHead[T any](head *DoublyNode[T], val T) *DoublyNode[T] {
    newNode := &DoublyNode[T]{
        Val:  val,
        Next: head,
    }
    
    if head != nil {
        head.Prev = newNode  // 原头节点的前驱指向新节点
    }
    
    return newNode
}
```

#### 在尾部插入节点

```go
func insertAtTail[T any](head *DoublyNode[T], val T) *DoublyNode[T] {
    newNode := &DoublyNode[T]{Val: val}
    
    if head == nil {
        return newNode
    }
    
    // 找到尾节点
    current := head
    for current.Next != nil {
        current = current.Next
    }
    
    current.Next = newNode
    newNode.Prev = current
    
    return head
}
```

#### 在指定位置插入节点

```go
func insertAfter[T any](node *DoublyNode[T], val T) {
    if node == nil {
        return
    }
    
    newNode := &DoublyNode[T]{
        Val:  val,
        Prev: node,
        Next: node.Next,
    }
    
    if node.Next != nil {
        node.Next.Prev = newNode
    }
    
    node.Next = newNode
}
```

#### 删除节点

```go
func deleteNode[T comparable](head *DoublyNode[T], val T) *DoublyNode[T] {
    if head == nil {
        return nil
    }
    
    // 删除头节点
    if head.Val == val {
        if head.Next != nil {
            head.Next.Prev = nil
        }
        return head.Next
    }
    
    // 查找并删除
    current := head
    for current != nil && current.Val != val {
        current = current.Next
    }
    
    if current != nil {
        // 修改前驱节点的 Next
        if current.Prev != nil {
            current.Prev.Next = current.Next
        }
        // 修改后继节点的 Prev
        if current.Next != nil {
            current.Next.Prev = current.Prev
        }
    }
    
    return head
}
```

#### 正向遍历

```go
func printForward[T any](head *DoublyNode[T]) {
    current := head
    fmt.Print("正向: nil <-> ")
    for current != nil {
        fmt.Printf("%v <-> ", current.Val)
        current = current.Next
    }
    fmt.Println("nil")
}
```

#### 反向遍历

```go
func printBackward[T any](head *DoublyNode[T]) {
    if head == nil {
        return
    }
    
    // 先找到尾节点
    tail := head
    for tail.Next != nil {
        tail = tail.Next
    }
    
    // 从尾节点反向遍历
    current := tail
    fmt.Print("反向: nil <-> ")
    for current != nil {
        fmt.Printf("%v <-> ", current.Val)
        current = current.Prev
    }
    fmt.Println("nil")
}
```

### 1.4 双向链表的优缺点

#### 优点
- ✅ 可以双向遍历（正向和反向）
- ✅ 删除节点时不需要从头查找前驱节点
- ✅ 在已知节点位置时，插入和删除操作更高效
- ✅ 某些算法实现更简单（如 LRU 缓存）

#### 缺点
- ❌ 每个节点需要额外的指针空间（多一个 Prev 指针）
- ❌ 插入和删除操作需要维护两个指针，代码更复杂
- ❌ 内存占用比单向链表多

### 1.5 应用场景

1. **浏览器的前进/后退功能**
2. **LRU（最近最少使用）缓存**
3. **文本编辑器的撤销/重做功能**
4. **音乐播放器的播放列表**
5. **需要频繁双向遍历的场景**

## 二、循环链表（Circular Linked List）

### 2.1 概念

循环链表是一种特殊的链表，其尾节点的 Next 指针指向头节点，形成一个环。

#### 单向循环链表

```
    ┌─────┬─────┐     ┌─────┬─────┐     ┌─────┬─────┐
    │ Val │Next │────>│ Val │Next │────>│ Val │Next │──┐
    │  1  │  ●  │     │  2  │  ●  │     │  3  │  ●  │  │
    └─────┴─────┘     └─────┴─────┘     └─────┴─────┘  │
       ↑                                                 │
       └─────────────────────────────────────────────────┘
```

#### 双向循环链表

```
    ┌─────┬─────┬─────┐     ┌─────┬─────┬─────┐     ┌─────┬─────┬─────┐
    │Prev │ Val │Next │────>│Prev │ Val │Next │────>│Prev │ Val │Next │──┐
    │  ●  │  1  │  ●  │     │  ●  │  2  │  ●  │     │  ●  │  3  │  ●  │  │
    └─────┴─────┴─────┘     └─────┴─────┴─────┘     └─────┴─────┴─────┘  │
       ↑  │                    ↑  │                    ↑  │                │
       │  └────────────────────┘  └────────────────────┘  │                │
       │                                                   │                │
       └───────────────────────────────────────────────────┴────────────────┘
```

### 2.2 节点结构定义

```go
// 单向循环链表节点
type CircularNode[T any] struct {
    Val  T
    Next *CircularNode[T]
}

// 双向循环链表节点
type DoublyCircularNode[T any] struct {
    Val  T
    Prev *DoublyCircularNode[T]
    Next *DoublyCircularNode[T]
}
```

### 2.3 基本操作（单向循环链表）

#### 创建循环链表

```go
func createCircularList[T any](values []T) *CircularNode[T] {
    if len(values) == 0 {
        return nil
    }
    
    head := &CircularNode[T]{Val: values[0]}
    current := head
    
    for i := 1; i < len(values); i++ {
        current.Next = &CircularNode[T]{Val: values[i]}
        current = current.Next
    }
    
    // 尾节点指向头节点，形成环
    current.Next = head
    
    return head
}
```

#### 在头部插入节点

```go
func insertAtHead[T any](head *CircularNode[T], val T) *CircularNode[T] {
    newNode := &CircularNode[T]{Val: val}
    
    if head == nil {
        newNode.Next = newNode  // 自己指向自己
        return newNode
    }
    
    // 找到尾节点
    tail := head
    for tail.Next != head {
        tail = tail.Next
    }
    
    newNode.Next = head
    tail.Next = newNode
    
    return newNode
}
```

#### 在尾部插入节点

```go
func insertAtTail[T any](head *CircularNode[T], val T) *CircularNode[T] {
    newNode := &CircularNode[T]{Val: val}
    
    if head == nil {
        newNode.Next = newNode
        return newNode
    }
    
    // 找到尾节点
    tail := head
    for tail.Next != head {
        tail = tail.Next
    }
    
    tail.Next = newNode
    newNode.Next = head
    
    return head
}
```

#### 删除节点

```go
func deleteNode[T comparable](head *CircularNode[T], val T) *CircularNode[T] {
    if head == nil {
        return nil
    }
    
    // 只有一个节点
    if head.Next == head {
        if head.Val == val {
            return nil
        }
        return head
    }
    
    // 删除头节点
    if head.Val == val {
        // 找到尾节点
        tail := head
        for tail.Next != head {
            tail = tail.Next
        }
        tail.Next = head.Next
        return head.Next
    }
    
    // 删除其他节点
    current := head
    for current.Next != head && current.Next.Val != val {
        current = current.Next
    }
    
    if current.Next.Val == val {
        current.Next = current.Next.Next
    }
    
    return head
}
```

#### 遍历循环链表

```go
func printCircularList[T any](head *CircularNode[T]) {
    if head == nil {
        fmt.Println("空链表")
        return
    }
    
    current := head
    fmt.Print("循环链表: ")
    
    // 第一次遍历
    fmt.Printf("%v -> ", current.Val)
    current = current.Next
    
    // 继续遍历直到回到头节点
    for current != head {
        fmt.Printf("%v -> ", current.Val)
        current = current.Next
    }
    
    fmt.Printf("(回到 %v)\n", head.Val)
}
```

#### 判断是否为循环链表

```go
// 使用快慢指针判断
func isCircular[T any](head *CircularNode[T]) bool {
    if head == nil {
        return false
    }
    
    slow := head
    fast := head
    
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        
        if slow == fast {
            return true
        }
    }
    
    return false
}
```

### 2.4 循环链表的优缺点

#### 优点
- ✅ 从任意节点都能访问到所有节点
- ✅ 适合需要循环遍历的场景
- ✅ 不需要判断 nil，遍历更简洁
- ✅ 某些算法实现更自然（如约瑟夫环问题）

#### 缺点
- ❌ 遍历时需要记录起点，避免死循环
- ❌ 插入和删除操作需要特殊处理头尾节点
- ❌ 调试困难，容易造成无限循环

### 2.5 应用场景

1. **约瑟夫环问题**（经典算法问题）
2. **操作系统的进程调度**（时间片轮转）
3. **多人游戏的回合制系统**
4. **循环播放列表**
5. **缓冲区管理**（环形缓冲区）

## 三、三种链表对比

| 特性 | 单向链表 | 双向链表 | 循环链表 |
|-----|---------|---------|---------|
| **节点指针数** | 1 个（Next） | 2 个（Prev, Next） | 1 或 2 个 |
| **内存占用** | 最小 | 最大 | 中等 |
| **正向遍历** | ✅ O(n) | ✅ O(n) | ✅ O(n) |
| **反向遍历** | ❌ 不支持 | ✅ O(n) | ⚠️ 需要记录起点 |
| **删除节点** | 需要前驱节点 | 直接删除 | 需要特殊处理 |
| **尾节点指向** | nil | nil | 头节点 |
| **实现复杂度** | 简单 | 中等 | 中等 |
| **适用场景** | 简单顺序访问 | 双向遍历 | 循环遍历 |

## 四、经典问题

### 4.1 约瑟夫环问题（循环链表）

n 个人围成一圈，从第 1 个人开始报数，报到 m 的人出列，然后从下一个人继续报数，求最后剩下的人。

```go
func josephus(n, m int) int {
    // 创建循环链表
    head := &CircularNode[int]{Val: 1}
    current := head
    
    for i := 2; i <= n; i++ {
        current.Next = &CircularNode[int]{Val: i}
        current = current.Next
    }
    current.Next = head  // 形成环
    
    // 开始报数
    prev := current
    current = head
    
    for current.Next != current {
        // 报数 m-1 次
        for i := 1; i < m; i++ {
            prev = current
            current = current.Next
        }
        
        // 删除当前节点
        fmt.Printf("出列: %d\n", current.Val)
        prev.Next = current.Next
        current = prev.Next
    }
    
    return current.Val
}
```

### 4.2 LRU 缓存（双向链表 + 哈希表）

```go
type LRUCache struct {
    capacity int
    cache    map[int]*DoublyNode[int]
    head     *DoublyNode[int]  // 虚拟头节点
    tail     *DoublyNode[int]  // 虚拟尾节点
}

func NewLRUCache(capacity int) *LRUCache {
    head := &DoublyNode[int]{}
    tail := &DoublyNode[int]{}
    head.Next = tail
    tail.Prev = head
    
    return &LRUCache{
        capacity: capacity,
        cache:    make(map[int]*DoublyNode[int]),
        head:     head,
        tail:     tail,
    }
}

// Get 获取值，并将节点移到头部（最近使用）
func (lru *LRUCache) Get(key int) int {
    if node, ok := lru.cache[key]; ok {
        lru.moveToHead(node)
        return node.Val
    }
    return -1
}

// Put 插入或更新值
func (lru *LRUCache) Put(key, value int) {
    if node, ok := lru.cache[key]; ok {
        node.Val = value
        lru.moveToHead(node)
    } else {
        newNode := &DoublyNode[int]{Val: value}
        lru.cache[key] = newNode
        lru.addToHead(newNode)
        
        if len(lru.cache) > lru.capacity {
            removed := lru.removeTail()
            delete(lru.cache, removed.Val)
        }
    }
}
```

## 五、总结

### 选择建议

- **单向链表**：简单场景，只需要顺序访问
- **双向链表**：需要双向遍历，或频繁删除节点
- **循环链表**：需要循环遍历，或实现轮转算法

### 时间复杂度对比

| 操作 | 单向链表 | 双向链表 | 循环链表 |
|-----|---------|---------|---------|
| 头部插入 | O(1) | O(1) | O(n)* |
| 尾部插入 | O(n) | O(1)** | O(n) |
| 头部删除 | O(1) | O(1) | O(n)* |
| 尾部删除 | O(n) | O(1)** | O(n) |
| 查找 | O(n) | O(n) | O(n) |

\* 循环链表需要找到尾节点  
\** 如果维护尾指针

### 实践建议

1. 优先使用单向链表，除非有明确需求
2. 需要双向遍历时使用双向链表
3. 需要循环遍历时使用循环链表
4. 考虑使用 Go 标准库的 `container/list`（双向链表实现）
