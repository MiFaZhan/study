# Go 链表操作：返回值 vs 指针的指针

## 两种实现方式对比

### 方式一：返回值方式（推荐 ✅）

```go
func deleteNode[T comparable](head *ListNode[T], val T) *ListNode[T] {
    if head == nil {
        return nil
    }
    if head.Val == val {
        return head.Next
    }
    // ... 其他逻辑
    return head
}

// 使用方式
head := createList([]int{1, 2, 3, 4, 5})
head = deleteNode(head, 3)  // 必须接收返回值
```

### 方式二：指针的指针方式

```go
func deleteNode[T comparable](head **ListNode[T], val T) {
    if *head == nil {
        return
    }
    if (*head).Val == val {
        *head = (*head).Next
        return
    }
    // ... 其他逻辑
}

// 使用方式
var head *ListNode[int]
createList(&head, []int{1, 2, 3, 4, 5})
deleteNode(&head, 3)  // 通过指针修改
```

## 详细对比

| 对比维度 | 返回值方式 | 指针的指针方式 |
|---------|-----------|---------------|
| **Go 惯用法** | ✅ 符合 Go 社区规范 | ❌ 更像 C 语言风格 |
| **代码可读性** | ✅ 清晰直观 | ❌ 语法复杂 `(*head).Val` |
| **意图表达** | ✅ 明确表示会产生新值 | ❌ 副作用隐藏在函数内部 |
| **安全性** | ✅ 编译器强制处理返回值 | ⚠️ 容易忘记传地址 |
| **标准库风格** | ✅ 与 `container/list` 一致 | ❌ 标准库不使用此方式 |
| **函数签名** | `func(head *T) *T` | `func(head **T)` |
| **调用方式** | `head = fn(head)` | `fn(&head)` |

## 优缺点分析

### 返回值方式

**优点：**
1. **符合 Go 语言习惯** - Go 推崇"返回新值"而非"修改参数"
2. **代码更清晰** - 一眼就能看出 `head` 被更新了
3. **类型安全** - 如果忘记赋值，编译器会警告未使用的返回值
4. **函数式风格** - 更接近纯函数，副作用明确
5. **易于测试** - 输入输出关系清晰

**缺点：**
1. 需要记得接收返回值（但这其实是优点，强制你处理结果）

### 指针的指针方式

**优点：**
1. 不需要返回值（但这在 Go 中不是优势）
2. 在 C 语言中很常见（但我们在写 Go）

**缺点：**
1. **语法复杂** - `**ListNode[T]` 和 `(*head).Val` 增加理解成本
2. **不符合 Go 惯例** - Go 标准库几乎不使用这种方式
3. **隐藏副作用** - 从函数签名看不出会修改参数
4. **容易出错** - 忘记传 `&head` 会导致编译错误
5. **调试困难** - 指针的指针增加了调试复杂度

## 实际案例：Go 标准库

Go 标准库中的链表操作都使用返回值方式：

```go
// container/list 包
func (l *List) PushFront(v any) *Element  // 返回新元素
func (l *List) Remove(e *Element) any     // 返回被删除的值

// strings 包
func Replace(s, old, new string, n int) string  // 返回新字符串

// append 内置函数
slice = append(slice, elem)  // 返回新切片
```

## 性能考虑

两种方式性能几乎相同：
- 返回值方式：返回一个指针（8 字节）
- 指针的指针方式：传递一个指针的地址（8 字节）

在链表操作中，真正的性能瓶颈是节点遍历，而不是参数传递方式。

## 结论

**推荐使用返回值方式** ✅

理由：
1. 这是 Go 语言的惯用法（idiomatic Go）
2. 代码更清晰、更安全、更易维护
3. 与 Go 标准库风格一致
4. 更容易被其他 Go 开发者理解和接受

**何时考虑指针的指针？**
- 在 C 语言中（因为 C 没有多返回值）
- 函数需要返回多个值，且其中一个是修改后的指针
- 与 C 库进行互操作时（cgo）

## 运行示例

```bash
# 运行返回值方式
go run return_value.go

# 运行指针的指针方式
go run pointer_to_pointer.go
```

两个程序输出相同的结果，但返回值方式的代码更符合 Go 语言的设计哲学。
