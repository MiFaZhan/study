# 链表查找操作详解

## 四种常见的查找方式

### 1. 按值查找 - 返回节点指针

```go
func findNode[T comparable](head *ListNode[T], val T) *ListNode[T] {
    current := head
    for current != nil {
        if current.Val == val {
            return current  // 找到返回节点指针
        }
        current = current.Next
    }
    return nil  // 未找到返回 nil
}
```

**使用场景：** 需要获取节点本身，以便后续操作（如修改、删除后继节点等）

**时间复杂度：** O(n)

**示例：**
```go
node := findNode(head, 4)
if node != nil {
    fmt.Printf("找到节点，值: %v\n", node.Val)
}
```

---

### 2. 按索引查找 - 返回节点指针

```go
func findByIndex[T any](head *ListNode[T], index int) *ListNode[T] {
    if index < 0 {
        return nil
    }
    current := head
    for i := 0; i < index && current != nil; i++ {
        current = current.Next
    }
    return current
}
```

**使用场景：** 根据位置访问节点（类似数组的下标访问）

**时间复杂度：** O(n)

**示例：**
```go
node := findByIndex(head, 2)  // 获取第 3 个节点（索引从 0 开始）
if node != nil {
    fmt.Printf("索引 2 的节点值: %v\n", node.Val)
}
```

---

### 3. 按值查找 - 返回索引位置

```go
func findIndex[T comparable](head *ListNode[T], val T) int {
    current := head
    index := 0
    for current != nil {
        if current.Val == val {
            return index  // 找到返回索引
        }
        current = current.Next
        index++
    }
    return -1  // 未找到返回 -1
}
```

**使用场景：** 需要知道元素在链表中的位置

**时间复杂度：** O(n)

**示例：**
```go
index := findIndex(head, 5)
if index != -1 {
    fmt.Printf("值 5 在索引位置: %d\n", index)
} else {
    fmt.Println("未找到该值")
}
```

---

### 4. 按条件查找 - 使用自定义函数

```go
func findByCondition[T any](head *ListNode[T], condition func(T) bool) *ListNode[T] {
    current := head
    for current != nil {
        if condition(current.Val) {
            return current
        }
        current = current.Next
    }
    return nil
}
```

**使用场景：** 复杂查找条件（如查找第一个大于某值的节点、查找符合特定规则的节点）

**时间复杂度：** O(n)

**示例：**
```go
// 查找第一个大于 3 的节点
node := findByCondition(head, func(val int) bool {
    return val > 3
})

// 查找第一个偶数节点
node := findByCondition(head, func(val int) bool {
    return val % 2 == 0
})

// 查找字符串长度大于 2 的节点
node := findByCondition(stringHead, func(val string) bool {
    return len(val) > 2
})
```

## 查找方式对比

| 查找方式 | 输入 | 输出 | 适用场景 | 泛型约束 |
|---------|------|------|---------|---------|
| `findNode` | 值 | 节点指针 | 需要操作节点本身 | `comparable` |
| `findByIndex` | 索引 | 节点指针 | 按位置访问 | `any` |
| `findIndex` | 值 | 索引位置 | 需要知道位置 | `comparable` |
| `findByCondition` | 条件函数 | 节点指针 | 复杂查找条件 | `any` |

## 性能分析

所有查找操作的时间复杂度都是 **O(n)**，因为链表必须从头开始遍历。

**与数组对比：**
- 数组按索引访问：O(1)
- 链表按索引访问：O(n)
- 数组按值查找：O(n)
- 链表按值查找：O(n)

**链表查找的特点：**
- 无法像数组那样直接通过索引快速访问
- 必须从头节点开始顺序遍历
- 适合顺序访问，不适合随机访问

## 优化技巧

### 1. 提前终止
```go
// 如果知道链表已排序，可以提前终止
func findInSortedList(head *ListNode[int], val int) *ListNode[int] {
    current := head
    for current != nil {
        if current.Val == val {
            return current
        }
        if current.Val > val {
            return nil  // 已经超过目标值，不可能找到
        }
        current = current.Next
    }
    return nil
}
```

### 2. 缓存常用节点
```go
// 如果频繁访问某些位置，可以缓存中间节点
type CachedList[T any] struct {
    head *ListNode[T]
    mid  *ListNode[T]  // 缓存中间节点
}
```

### 3. 使用哈希表加速
```go
// 如果需要频繁按值查找，可以维护一个哈希表
type FastSearchList[T comparable] struct {
    head *ListNode[T]
    nodeMap map[T]*ListNode[T]  // 值到节点的映射
}
```

## 实际应用场景

1. **按值查找：** 检查链表中是否存在某个元素
2. **按索引查找：** 实现类似数组的访问接口
3. **查找索引：** 判断元素在链表中的位置关系
4. **按条件查找：** 
   - 查找第一个满足条件的节点
   - 过滤链表元素
   - 实现复杂的业务逻辑
