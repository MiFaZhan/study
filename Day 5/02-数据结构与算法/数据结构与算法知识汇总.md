# 数据结构与算法知识汇总

本文系统梳理数据结构与算法的核心知识，涵盖常用数据结构（数组、链表、哈希表、树）和经典算法（排序、查找），帮助快速建立知识框架。

## 一、基础数据结构

### 1. 数组（Array）
#### 1.1 基本概念
- **定义**：一种线性数据结构，在内存中连续存储相同类型的元素
- **特点**：
  - 支持随机访问，通过索引直接定位元素，时间复杂度 O(1)
  - 固定大小（静态数组）或动态扩容（动态数组）
  - 插入/删除元素需要移动其他元素，时间复杂度 O(n)

#### 1.2 时间复杂度
| 操作 | 时间复杂度 |
|------|-----------|
| 访问 | O(1) |
| 查找 | O(n) |
| 插入（末尾） | O(1) |
| 插入（中间） | O(n) |
| 删除 | O(n) |

#### 1.3 Go 语言实现
```go
// 静态数组
var arr [5]int = [5]int{1, 2, 3, 4, 5}

// 动态数组（切片）
slice := []int{1, 2, 3, 4, 5}
slice = append(slice, 6)  // 追加元素

// 访问元素
fmt.Println(slice[0])  // 1

// 遍历
for i, v := range slice {
    fmt.Printf("索引: %d, 值: %d\n", i, v)
}
```

#### 1.4 应用场景
- 需要频繁随机访问元素
- 数据量固定或变化不大
- 实现其他数据结构的基础（栈、队列、堆等）

### 2. 链表（Linked List）

#### 2.1 基本概念
- **定义**：由节点组成的线性数据结构，每个节点包含数据和指向下一个节点的指针
- **类型**：
  - **单向链表**：每个节点只有一个指向下一个节点的指针
  - **双向链表**：每个节点有两个指针，分别指向前一个和后一个节点
  - **循环链表**：尾节点指向头节点，形成环

#### 2.2 特点
- 不需要连续内存空间
- 插入/删除操作高效（只需修改指针），时间复杂度 O(1)
- 不支持随机访问，查找需要遍历，时间复杂度 O(n)
- 需要额外空间存储指针

#### 2.3 时间复杂度
| 操作 | 时间复杂度 |
|------|-----------|
| 访问 | O(n) |
| 查找 | O(n) |
| 插入（已知位置） | O(1) |
| 删除（已知位置） | O(1) |

#### 2.4 Go 语言实现
```go
// 定义链表节点
type ListNode struct {
    Val  int
    Next *ListNode
}

// 创建链表
func createList(values []int) *ListNode {
    if len(values) == 0 {
        return nil
    }
    head := &ListNode{Val: values[0]}
    current := head
    for i := 1; i < len(values); i++ {
        current.Next = &ListNode{Val: values[i]}
        current = current.Next
    }
    return head
}

// 遍历链表
func printList(head *ListNode) {
    current := head
    for current != nil {
        fmt.Printf("%d -> ", current.Val)
        current = current.Next
    }
    fmt.Println("nil")
}

// 插入节点（在头部）
func insertAtHead(head *ListNode, val int) *ListNode {
    newNode := &ListNode{Val: val, Next: head}
    return newNode
}

// 删除节点
func deleteNode(head *ListNode, val int) *ListNode {
    if head == nil {
        return nil
    }
    if head.Val == val {
        return head.Next
    }
    current := head
    for current.Next != nil && current.Next.Val != val {
        current = current.Next
    }
    if current.Next != nil {
        current.Next = current.Next.Next
    }
    return head
}
```

#### 2.5 应用场景
- 频繁插入/删除操作
- 不需要随机访问
- 实现栈、队列、LRU 缓存等

### 3. 哈希表（Hash Table）

#### 3.1 基本概念
- **定义**：通过哈希函数将键映射到数组索引，实现快速查找的数据结构
- **组成**：
  - **哈希函数**：将键转换为数组索引
  - **数组**：存储键值对
  - **冲突解决**：处理不同键映射到相同索引的情况

#### 3.2 冲突解决方法
- **链地址法（Chaining）**：每个数组位置存储一个链表，冲突的元素添加到链表中
- **开放寻址法（Open Addressing）**：冲突时寻找下一个空位置
  - 线性探测：依次检查下一个位置
  - 二次探测：按平方数间隔探测
  - 双重哈希：使用第二个哈希函数

#### 3.3 时间复杂度
| 操作 | 平均时间 | 最坏时间 |
|------|---------|---------|
| 查找 | O(1) | O(n) |
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |

#### 3.4 Go 语言实现
```go
// Go 内置的 map 就是哈希表
hashMap := make(map[string]int)

// 插入
hashMap["apple"] = 5
hashMap["banana"] = 3

// 查找
value, exists := hashMap["apple"]
if exists {
    fmt.Println("apple:", value)
}

// 删除
delete(hashMap, "banana")

// 遍历
for key, value := range hashMap {
    fmt.Printf("%s: %d\n", key, value)
}

// 检查键是否存在
if _, ok := hashMap["orange"]; !ok {
    fmt.Println("orange 不存在")
}
```

#### 3.5 应用场景
- 快速查找、插入、删除
- 去重
- 统计频率
- 缓存实现
- 数据库索引

### 4. 树（Tree）

#### 4.1 基本概念
- **定义**：由节点组成的层次结构，每个节点有零个或多个子节点
- **术语**：
  - **根节点**：树的顶部节点
  - **叶子节点**：没有子节点的节点
  - **父节点/子节点**：直接连接的上下层节点
  - **深度**：从根到节点的边数
  - **高度**：从节点到最深叶子的边数
  - **层级**：根节点为第 0 层

#### 4.2 二叉树（Binary Tree）
- **定义**：每个节点最多有两个子节点（左子节点和右子节点）
- **类型**：
  - **满二叉树**：每层节点都达到最大数量
  - **完全二叉树**：除最后一层外都是满的，最后一层从左到右连续
  - **平衡二叉树**：左右子树高度差不超过 1

#### 4.3 二叉搜索树（BST - Binary Search Tree）
- **特点**：
  - 左子树所有节点值 < 根节点值
  - 右子树所有节点值 > 根节点值
  - 左右子树也是二叉搜索树
- **时间复杂度**：
  - 平均：O(log n)
  - 最坏（退化成链表）：O(n)

#### 4.4 Go 语言实现
```go
// 定义二叉树节点
type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}

// 创建节点
func newNode(val int) *TreeNode {
    return &TreeNode{Val: val}
}

// 二叉搜索树插入
func insertBST(root *TreeNode, val int) *TreeNode {
    if root == nil {
        return newNode(val)
    }
    if val < root.Val {
        root.Left = insertBST(root.Left, val)
    } else if val > root.Val {
        root.Right = insertBST(root.Right, val)
    }
    return root
}

// 二叉搜索树查找
func searchBST(root *TreeNode, val int) *TreeNode {
    if root == nil || root.Val == val {
        return root
    }
    if val < root.Val {
        return searchBST(root.Left, val)
    }
    return searchBST(root.Right, val)
}

// 前序遍历（根-左-右）
func preorderTraversal(root *TreeNode) {
    if root == nil {
        return
    }
    fmt.Print(root.Val, " ")
    preorderTraversal(root.Left)
    preorderTraversal(root.Right)
}

// 中序遍历（左-根-右）- BST 中序遍历结果是有序的
func inorderTraversal(root *TreeNode) {
    if root == nil {
        return
    }
    inorderTraversal(root.Left)
    fmt.Print(root.Val, " ")
    inorderTraversal(root.Right)
}

// 后序遍历（左-右-根）
func postorderTraversal(root *TreeNode) {
    if root == nil {
        return
    }
    postorderTraversal(root.Left)
    postorderTraversal(root.Right)
    fmt.Print(root.Val, " ")
}

// 层序遍历（广度优先）
func levelOrderTraversal(root *TreeNode) {
    if root == nil {
        return
    }
    queue := []*TreeNode{root}
    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        fmt.Print(node.Val, " ")
        if node.Left != nil {
            queue = append(queue, node.Left)
        }
        if node.Right != nil {
            queue = append(queue, node.Right)
        }
    }
}
```

- 文件系统目录结构
- 数据库索引（B树、B+树）
- 表达式解析
- 决策树
- 路由表

## 二、常见算法

### 1. 排序算法

#### 1.1 冒泡排序（Bubble Sort）
- **原理**：重复遍历数组，比较相邻元素并交换，将最大元素"冒泡"到末尾
- **时间复杂度**：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：稳定

```go
func bubbleSort(arr []int) {
    n := len(arr)
    for i := 0; i < n-1; i++ {
        swapped := false
        for j := 0; j < n-i-1; j++ {
            if arr[j] > arr[j+1] {
                arr[j], arr[j+1] = arr[j+1], arr[j]
                swapped = true
            }
        }
        if !swapped {  // 优化：如果没有交换，说明已排序
            break
        }
    }
}
```

#### 1.2 选择排序（Selection Sort）
- **原理**：每次从未排序部分选择最小元素，放到已排序部分末尾
- **时间复杂度**：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：不稳定

```go
func selectionSort(arr []int) {
    n := len(arr)
    for i := 0; i < n-1; i++ {
        minIdx := i
        for j := i + 1; j < n; j++ {
            if arr[j] < arr[minIdx] {
                minIdx = j
            }
        }
        arr[i], arr[minIdx] = arr[minIdx], arr[i]
    }
}
```

#### 1.3 插入排序（Insertion Sort）
- **原理**：将元素插入到已排序部分的正确位置
- **时间复杂度**：O(n²)，最好情况 O(n)
- **空间复杂度**：O(1)
- **稳定性**：稳定

```go
func insertionSort(arr []int) {
    n := len(arr)
    for i := 1; i < n; i++ {
        key := arr[i]
        j := i - 1
        // 将大于 key 的元素向后移动
        for j >= 0 && arr[j] > key {
            arr[j+1] = arr[j]
            j--
        }
        arr[j+1] = key
    }
}
```

#### 1.4 快速排序（Quick Sort）
- **原理**：选择基准元素，将数组分为小于和大于基准的两部分，递归排序
- **时间复杂度**：平均 O(n log n)，最坏 O(n²)
- **空间复杂度**：O(log n)
- **稳定性**：不稳定

```go
func quickSort(arr []int, low, high int) {
    if low < high {
        pi := partition(arr, low, high)
        quickSort(arr, low, pi-1)
        quickSort(arr, pi+1, high)
    }
}

func partition(arr []int, low, high int) int {
    pivot := arr[high]  // 选择最后一个元素作为基准
    i := low - 1
    for j := low; j < high; j++ {
        if arr[j] < pivot {
            i++
            arr[i], arr[j] = arr[j], arr[i]
        }
    }
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1
}
```

#### 1.5 归并排序（Merge Sort）
- **原理**：分治法，将数组分成两半，递归排序后合并
- **时间复杂度**：O(n log n)
- **空间复杂度**：O(n)
- **稳定性**：稳定

```go
func mergeSort(arr []int) []int {
    if len(arr) <= 1 {
        return arr
    }
    mid := len(arr) / 2
    left := mergeSort(arr[:mid])
    right := mergeSort(arr[mid:])
    return merge(left, right)
}

func merge(left, right []int) []int {
    result := make([]int, 0, len(left)+len(right))
    i, j := 0, 0
    for i < len(left) && j < len(right) {
        if left[i] <= right[j] {
            result = append(result, left[i])
            i++
        } else {
            result = append(result, right[j])
            j++
        }
    }
    result = append(result, left[i:]...)
    result = append(result, right[j:]...)
    return result
}
```

#### 1.6 排序算法对比

| 算法 | 时间复杂度（平均） | 时间复杂度（最坏） | 空间复杂度 | 稳定性 |
|------|------------------|------------------|-----------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |

### 2. 查找算法

#### 2.1 线性查找（Linear Search）
- **原理**：从头到尾遍历数组，逐个比较
- **时间复杂度**：O(n)
- **适用**：无序数组

```go
func linearSearch(arr []int, target int) int {
    for i, v := range arr {
        if v == target {
            return i
        }
    }
    return -1  // 未找到
}
```

#### 2.2 二分查找（Binary Search）
- **原理**：在有序数组中，每次比较中间元素，缩小一半搜索范围
- **时间复杂度**：O(log n)
- **前提**：数组必须有序

```go
// 迭代实现
func binarySearch(arr []int, target int) int {
    left, right := 0, len(arr)-1
    for left <= right {
        mid := left + (right-left)/2  // 防止溢出
        if arr[mid] == target {
            return mid
        } else if arr[mid] < target {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    return -1
}

// 递归实现
func binarySearchRecursive(arr []int, target, left, right int) int {
    if left > right {
        return -1
    }
    mid := left + (right-left)/2
    if arr[mid] == target {
        return mid
    } else if arr[mid] < target {
        return binarySearchRecursive(arr, target, mid+1, right)
    } else {
        return binarySearchRecursive(arr, target, left, mid-1)
    }
}
```

#### 2.3 二分查找变体

```go
// 查找第一个等于目标值的位置
func binarySearchFirst(arr []int, target int) int {
    left, right := 0, len(arr)-1
    result := -1
    for left <= right {
        mid := left + (right-left)/2
        if arr[mid] == target {
            result = mid
            right = mid - 1  // 继续向左查找
        } else if arr[mid] < target {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    return result
}

// 查找最后一个等于目标值的位置
func binarySearchLast(arr []int, target int) int {
    left, right := 0, len(arr)-1
    result := -1
    for left <= right {
        mid := left + (right-left)/2
        if arr[mid] == target {
            result = mid
            left = mid + 1  // 继续向右查找
        } else if arr[mid] < target {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    return result
}

// 查找第一个大于等于目标值的位置
func binarySearchLowerBound(arr []int, target int) int {
    left, right := 0, len(arr)
    for left < right {
        mid := left + (right-left)/2
        if arr[mid] < target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    return left
}
```

## 三、算法思想

### 1. 递归（Recursion）
- **定义**：函数调用自身解决问题
- **要素**：
  - 基准情况（Base Case）：递归终止条件
  - 递归情况（Recursive Case）：问题分解

```go
// 阶乘
func factorial(n int) int {
    if n <= 1 {
        return 1
    }
    return n * factorial(n-1)
}

// 斐波那契数列
func fibonacci(n int) int {
    if n <= 1 {
        return n
    }
    return fibonacci(n-1) + fibonacci(n-2)
}
```

### 2. 分治法（Divide and Conquer）
- **思想**：将问题分解为更小的子问题，递归解决后合并结果
- **步骤**：
  1. 分解：将问题分成若干子问题
  2. 解决：递归解决子问题
  3. 合并：将子问题的解合并为原问题的解
- **典型应用**：归并排序、快速排序、二分查找

### 3. 动态规划（Dynamic Programming）
- **思想**：将复杂问题分解为重叠子问题，存储子问题的解避免重复计算
- **特征**：
  - 最优子结构：问题的最优解包含子问题的最优解
  - 重叠子问题：子问题会被多次求解
- **方法**：
  - 自顶向下（记忆化搜索）
  - 自底向上（表格法）

```go
// 斐波那契数列（动态规划）
func fibonacciDP(n int) int {
    if n <= 1 {
        return n
    }
    dp := make([]int, n+1)
    dp[0], dp[1] = 0, 1
    for i := 2; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }
    return dp[n]
}

// 空间优化版本
func fibonacciOptimized(n int) int {
    if n <= 1 {
        return n
    }
    prev, curr := 0, 1
    for i := 2; i <= n; i++ {
        prev, curr = curr, prev+curr
    }
    return curr
}
```

### 4. 贪心算法（Greedy Algorithm）
- **思想**：每步选择当前最优解，期望得到全局最优解
- **特点**：
  - 不一定得到全局最优解
  - 效率高，实现简单
- **适用条件**：
  - 贪心选择性质
  - 最优子结构

```go
// 找零钱问题（贪心）
func coinChange(coins []int, amount int) []int {
    sort.Sort(sort.Reverse(sort.IntSlice(coins)))  // 降序排序
    result := []int{}
    for _, coin := range coins {
        for amount >= coin {
            result = append(result, coin)
            amount -= coin
        }
    }
    return result
}
```

## 四、时间复杂度与空间复杂度

### 1. 时间复杂度
衡量算法执行时间随输入规模增长的趋势。

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 数组访问、哈希表查找 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 归并排序、快速排序 |
| O(n²) | 平方 | 冒泡排序、选择排序 |
| O(2ⁿ) | 指数 | 递归斐波那契 |
| O(n!) | 阶乘 | 全排列 |

### 2. 空间复杂度
衡量算法所需额外内存空间随输入规模增长的趋势。

## 总结

掌握数据结构与算法是编程的基本功。数组适合随机访问，链表适合频繁插入删除，哈希表提供快速查找，树结构适合层次化数据。排序算法中，快速排序和归并排序效率高，二分查找要求数据有序。理解时间复杂度和空间复杂度，能帮助选择合适的算法和数据结构，写出高效的代码。
