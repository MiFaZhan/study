# GoFrame 数据流转过程 - 精简版

## 示例 1：登录请求（查询数据库）

### 整体架构流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                     HTTP 请求（登录）                             │
│              POST /v1/users/login                                │
│              {"username":"oldme", "password":"123456"}           │
└────────────────────────────┬────────────────────────────────────┘
                             │
                    ┌────────▼────────┐
                    │   main.go       │  程序入口
                    │  - 初始化 i18n   │  • g.I18n().SetLanguage("zh-CN")
                    │  - 连接数据库    │  • g.DB().PingMaster()
                    │  - 启动服务      │  • cmd.Main.Run()
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   cmd 层        │  路由注册
                    │  cmd.go         │  • g.Server() 创建服务器
                    │                 │  • group.Middleware() 注册中间件
                    │                 │  • group.Bind(users.NewV1())
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  路由匹配        │  GoFrame 路由系统
                    │                 │  • 根据 g.Meta 中的 path 匹配
                    │                 │  • 找到 users.Login() 方法
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  中间件层        │  统一处理
                    │  Response       │  • MiddlewareHandlerResponse
                    │                 │  • 统一响应格式
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  参数处理        │  自动化
                    │                 │  • JSON → LoginReq 结构体
                    │                 │  • 验证规则（v 标签）
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  API 层         │  接口定义
                    │  LoginReq       │  • Username, Password
                    │  LoginRes       │  • Token
                    │  g.Meta 标签    │  • path:"users/login" method:"post"
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ Controller 层   │  请求调度
                    │ users_v1_login  │  • 接收 LoginReq
                    │                 │  • 调用 c.users.Login()
                    │                 │  • 返回 LoginRes{Token}
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  Logic 层       │  业务逻辑
                    │ users_login.go  │  • dao.Users.Where().Scan(&user)
                    │                 │  • 验证密码（MD5 加密比对）
                    │                 │  • 生成 JWT Token
                    │                 │  • 返回 token 字符串
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │   DAO 层        │  数据访问
                    │  dao.Users      │  • Ctx(ctx).Where("username", username)
                    │                 │  • Scan(&user) 查询用户
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  Model 层       │  数据模型
                    │  entity.Users   │  • 接收查询结果
                    │                 │  • Id, Username, Password, Email
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │    数据库        │  MySQL
                    │  SELECT * FROM  │  • 执行查询
                    │  users WHERE    │  • 返回用户记录
                    │  username=?     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  逐层返回        │  返回链路
                    │  DAO → Logic    │  • 返回 entity.Users
                    │  Logic → Ctrl   │  • 返回 token 字符串
                    │  Ctrl → 中间件  │  • 返回 LoginRes{Token}
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  HTTP 响应      │  统一格式
                    │  {              │  • code: 0（成功）
                    │    "code": 0,   │  • message: ""
                    │    "message":"",│  • data: {"token": "eyJ..."}
                    │    "data": {    │
                    │      "token":"..│
                    │    }            │
                    │  }              │
                    └─────────────────┘
```

## 核心概念

### 分层架构
- **API 层**：定义接口契约（路由、参数、验证）
- **Controller 层**：接收请求，调度业务逻辑
- **Logic 层**：实现业务逻辑，调用 DAO
- **DAO 层**：封装数据库操作
- **Model 层**：定义数据模型（Entity、DO）

### 关键特性
- **自动路由注册**：通过 g.Meta 标签 + Bind() 实现
- **自动参数绑定**：JSON 自动解析为结构体
- **自动参数验证**：根据 v 标签验证参数
- **自动响应包装**：统一的响应格式
- **依赖注入**：Controller 持有 Logic 引用
- **链式调用**：DAO 层提供流畅的 API

### 数据流向
```
请求 → 路由 → 中间件 → Controller → Logic → DAO → 数据库
                                                    ↓
响应 ← 中间件 ← Controller ← Logic ← DAO ← 数据库
```

## 两个示例的关键区别

### 登录请求（查询操作）
- **无需鉴权**：公开接口
- **数据库操作**：SELECT 查询
- **业务逻辑**：密码验证 + JWT 生成
- **返回数据**：Token 字符串

### 创建单词（修改操作）
- **需要鉴权**：JWT Token 验证
- **数据库操作**：SELECT（检查重复）+ INSERT（插入）
- **业务逻辑**：重复检查 + 数据插入
- **返回数据**：空（只返回成功/失败）

## 核心概念总结

### 分层架构
```
API 层 → Controller 层 → Logic 层 → DAO 层 → Model 层 → 数据库
```

### 各层职责
- **API 层**：定义接口契约（路由、参数、验证）
- **Controller 层**：接收请求，调度业务逻辑
- **Logic 层**：实现业务逻辑，调用 DAO
- **DAO 层**：封装数据库操作
- **Model 层**：定义数据模型（Entity、DO）

### 关键特性
1. **自动路由注册**：通过 g.Meta 标签 + Bind() 实现
2. **自动参数绑定**：JSON 自动解析为结构体
3. **自动参数验证**：根据 v 标签验证参数
4. **自动响应包装**：统一的响应格式
5. **依赖注入**：Controller 持有 Logic 引用
6. **链式调用**：DAO 层提供流畅的 API

### 数据流向
```
请求 → 路由 → 中间件 → Controller → Logic → DAO → 数据库
                                                    ↓
响应 ← 中间件 ← Controller ← Logic ← DAO ← 数据库
```

### 中间件执行顺序
```
请求 → MiddlewareHandlerResponse → Auth（如果需要）→ Controller
                                                        ↓
响应 ← MiddlewareHandlerResponse ← Auth（如果需要）← Controller
```

### Model 层的两种对象
- **Entity**：表实体，用于接收查询结果（具体类型）
- **DO**：操作对象，用于构建查询条件和更新数据（any 类型）

### DAO 层常用方法
```go
dao.Users.Ctx(ctx)              // 传入上下文
  .Where("field", value)        // 查询条件
  .WhereLike("field", pattern)  // 模糊查询
  .Data(do.Users{...})          // 设置数据
  .Insert()                     // 插入
  .Update()                     // 更新
  .Delete()                     // 删除
  .Scan(&entity)                // 查询单条
  .All()                        // 查询多条
  .Count()                      // 统计数量
  .Page(page, size)             // 分页查询
```
