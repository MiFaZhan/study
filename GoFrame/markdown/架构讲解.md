## 1. API 层 - 接口定义

**位置**：`api/users/v1/users.go`

```go
package v1

import "github.com/gogf/gf/v2/frame/g"

type RegisterReq struct {
    g.Meta   `path:"users/register" method:"post"`
    Username string `json:"username"`
    Password string `json:"password"`
    Email    string `json:"email"`
}

type RegisterRes struct {}
```

**作用**：
- 定义接口的"契约"（路由、方法、参数）
- `g.Meta` 是 GoFrame 的魔法，通过 struct tag 声明式定义接口属性
- `path:"users/register"` - 接口路径
- `method:"post"` - HTTP 方法
- 请求参数通过 JSON tag 自动映射

**自动生成的接口文件**：`api/users/users.go`

```go
package users

type IUsersV1 interface {
    Register(ctx context.Context, req *v1.RegisterReq) (res *v1.RegisterRes, err error)
}
```

这是 GoFrame 根据你的 API 定义自动生成的接口，Controller 必须实现这个接口。

## 2. Controller 层 - 控制器

**位置**：`internal/controller/users/`

### 文件 1：`users_new.go` - 控制器定义

```go
package users

import (
    "star/api/users"
    userLogic "star/internal/logic/users"
)

type ControllerV1 struct {
    users *userLogic.Users  // 依赖注入：持有 Logic 层实例
}

func NewV1() users.IUsersV1 {
    return &ControllerV1{
        users: userLogic.New(),  // 初始化依赖
    }
}
```

**关键概念**：

1. **依赖注入**：
   - `users *userLogic.Users` 字段就是注入的依赖
   - 通过结构体字段持有 Logic 层实例
   - 不是在方法内部 `new`，而是在创建 Controller 时注入

2. **工厂模式**：
   - `NewV1()` 是工厂函数，负责创建和初始化 Controller
   - 返回接口类型 `users.IUsersV1`，而不是具体类型
   - 这是面向接口编程的体现

### 文件 2：`users_v1_register.go` - 具体方法实现

```go
package users

import (
    "context"
    "star/api/users/v1"
)

func (c *ControllerV1) Register(ctx context.Context, req *v1.RegisterReq) (res *v1.RegisterRes, err error) {
    err = c.users.Register(ctx, req.Username, req.Password, req.Email)
    return nil, err
}
```

**职责**：
- 接收 HTTP 请求（已经被框架解析为 `RegisterReq` 结构体）
- 调用 Logic 层的方法：`c.users.Register()`
- 返回响应（框架会自动序列化为 JSON）

**为什么能调用 `c.users.Register()`？**
因为在 `users_new.go` 中，`ControllerV1` 结构体有 `users` 字段！

## 3. Logic 层 - 业务逻辑

**位置**：`internal/logic/users/`

### 文件 1：`users.go` - Logic 对象定义

```go
package users

type Users struct {}

func New() *Users {
    return &Users{}
}
```

定义业务逻辑对象，可以在这里添加字段（比如缓存、配置等）。

### 文件 2：`users_register.go` - 注册逻辑

```go
package users

import (
    "context"
    "star/internal/dao"
    "star/internal/model/do"
)

func (u *Users) Register(ctx context.Context, username, password, email string) error {
    _, err := dao.Users.Ctx(ctx).Data(do.Users{
        Username: username,
        Password: password,
        Email:    email,
    }).Insert()
    if err != nil {
        return err
    }
    return nil
}
```

**职责**：
- 处理业务逻辑（这里比较简单，直接插入数据库）
- 调用 DAO 层操作数据库
- 可以在这里添加：
  - 数据验证
  - 密码加密
  - 发送邮件
  - 调用其他服务
  - 事务处理

## 5. CMD 层 - 启动和路由注册

**位置**：`internal/cmd/cmd.go`

```go
package cmd

import (
    "star/internal/controller/hello"
    "star/internal/controller/users"
)

var (
    Main = gcmd.Command{
        Func: func(ctx context.Context, parser *gcmd.Parser) (err error) {
            s := g.Server()
            s.Group("/", func(group *ghttp.RouterGroup) {
                group.Middleware(ghttp.MiddlewareHandlerResponse)
                
                // 注册 hello 控制器
                group.Bind(hello.NewV1())
                
                // 注册 users 控制器到 /v1 分组
                group.Group("/v1", func(group *ghttp.RouterGroup) {
                    group.Bind(users.NewV1())
                })
            })
            s.Run()
            return nil
        },
    }
)
```

**关键点**：

1. **分组路由**：
   - `group.Group("/v1", ...)` 创建 `/v1` 路由分组
   - 所有在这个分组内注册的路由都会加上 `/v1` 前缀

2. **控制器绑定**：
   - `group.Bind(users.NewV1())` 绑定控制器
   - GoFrame 会自动扫描控制器的方法，根据 API 定义生成路由

3. **中间件**：
   - `group.Middleware(ghttp.MiddlewareHandlerResponse)` 
   - 统一处理响应格式，自动包装成 `{code, message, data}` 格式

## 完整的请求流程

假设发起请求：
```bash
curl -X POST http://127.0.0.1:8000/v1/users/register \
  -H "Content-Type: application/json" \
  -d '{"username":"oldme", "password":"123456", "email":"test@example.com"}'
```

**流程详解**：

```
1. HTTP POST /v1/users/register
   ↓
2. GoFrame 路由系统匹配
   - 找到 /v1 分组
   - 找到 users 控制器
   - 根据 g.Meta 中的 path:"users/register" 匹配到 Register 方法
   ↓
3. 中间件处理
   - MiddlewareHandlerResponse 拦截请求
   ↓
4. 参数解析
   - JSON 自动解析为 RegisterReq 结构体
   - {"username":"oldme"} → req.Username = "oldme"
   ↓
5. 调用 Controller
   - ControllerV1.Register(ctx, req)
   ↓
6. Controller 调用 Logic
   - c.users.Register(ctx, req.Username, req.Password, req.Email)
   - 这里的 c.users 是在 NewV1() 中注入的
   ↓
7. Logic 调用 DAO
   - dao.Users.Ctx(ctx).Data(do.Users{...}).Insert()
   ↓
8. DAO 执行 SQL
   - INSERT INTO users (username, password, email) VALUES (...)
   ↓
9. 返回结果
   - Logic 返回 error 给 Controller
   - Controller 返回 (nil, err)
   ↓
10. 中间件包装响应
    - 自动包装成 {"code":0, "message":"", "data":null}
    ↓
11. 返回 JSON 给客户端
```

## 核心设计模式

### 1. 依赖注入（Dependency Injection）

```go
type ControllerV1 struct {
    users *userLogic.Users  // 注入的依赖
}

func NewV1() users.IUsersV1 {
    return &ControllerV1{
        users: userLogic.New(),  // 在这里注入
    }
}
```

**好处**：
- 解耦：Controller 不需要知道 Logic 如何创建
- 可测试：可以注入 mock 对象进行单元测试
- 灵活：可以轻松替换实现

### 2. 工厂模式（Factory Pattern）

```go
func NewV1() users.IUsersV1 {
    return &ControllerV1{...}
}
```

**好处**：
- 封装创建逻辑
- 返回接口而不是具体类型
- 便于扩展和维护

### 3. 面向接口编程

```go
// 自动生成的接口
type IUsersV1 interface {
    Register(ctx context.Context, req *v1.RegisterReq) (res *v1.RegisterRes, err error)
}

// Controller 实现接口
type ControllerV1 struct {...}
func (c *ControllerV1) Register(...) {...}
```

**好处**：
- 规范约束：必须实现接口定义的方法
- 类型安全：编译时检查
- 便于替换实现

## 为什么这样设计？

### 单一职责原则
- API 层：只管接口定义
- Controller 层：只管请求调度
- Logic 层：只管业务逻辑
- DAO 层：只管数据访问

### 开闭原则
- 要加新功能？在 Logic 层添加方法
- 要改数据库？只改 DAO 层
- 要改接口格式？只改 API 层

### 可测试性
```go
// 可以轻松 mock Logic 层进行测试
type MockUsers struct {}
func (m *MockUsers) Register(...) error {
    return nil  // 模拟成功
}

controller := &ControllerV1{
    users: &MockUsers{},  // 注入 mock 对象
}
```
