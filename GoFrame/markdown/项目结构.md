这是一个基于 GoFrame v2.7.1 的标准单体应用（SingleRepo）项目，采用了经典的分层架构设计。

### 分层架构详解

#### 1. API 层 (api/)
**职责：** 定义对外的 API 接口契约

**文件组织：**
- `api/hello/hello.go`：接口定义（自动生成，不可手动修改）
  ```go
  type IHelloV1 interface {
      Hello(ctx context.Context, req *v1.HelloReq) (res *v1.HelloRes, err error)
  }
  ```
- `api/hello/v1/hello.go`：请求/响应结构体 + 路由元信息
  ```go
  type HelloReq struct {
      g.Meta `path:"/hello" tags:"Hello" method:"get" summary:"You first hello api"`
  }
  type HelloRes struct {
      g.Meta `mime:"text/html" example:"string"`
  }
  ```

**特点：**
- 使用 `g.Meta` 标签定义路由、HTTP 方法、文档说明
- 支持版本管理（v1, v2, v3...）
- 通过 `gf gen ctrl` 命令自动生成接口定义和 Controller 骨架

---

#### 2. Controller 层 (internal/controller/)
**职责：** 接收 HTTP 请求，调用业务逻辑，返回响应

**文件组织：**
- `hello.go`：手动编写的业务代码（可自由修改）
- `hello_new.go`：自动生成的构造函数（不可修改）
  ```go
  func NewV1() hello.IHelloV1 {
      return &ControllerV1{}
  }
  ```
- `hello_v1_hello.go`：自动生成的接口实现骨架（可修改）
  ```go
  func (c *ControllerV1) Hello(ctx context.Context, req *v1.HelloReq) (res *v1.HelloRes, err error) {
      g.RequestFromCtx(ctx).Response.Writeln("Hello World!")
      return
  }
  ```

**特点：**
- 实现 API 层定义的接口
- 通过 `cmd.go` 中的 `group.Bind()` 自动注册路由
- 使用 `MiddlewareHandlerResponse` 中间件统一处理响应格式

---

#### 3. Logic 层 (internal/logic/)
**职责：** 实现复杂的业务逻辑

**使用场景：**
- 复杂的业务规则处理
- 多表关联查询和数据组装
- 事务处理
- 跨模块的业务协调

**特点：**
- 实现 Service 层定义的接口
- 可被多个 Controller 复用
- 不直接处理 HTTP 请求，专注于业务逻辑
- 通过 `gf gen service` 自动生成接口绑定代码

---

#### 4. Service 层 (internal/service/)
**职责：** 定义业务服务的接口（interface）

**设计原则：**
- 只包含方法签名，不包含具体实现
- 遵循"面向接口编程"原则
- Controller 依赖 Service 接口，而非直接依赖 Logic 实现
- 便于单元测试和依赖注入

**生成方式：**
```bash
make service  # 或 gf gen service
```

---

#### 5. DAO 层 (internal/dao/)
**职责：** 数据访问对象，封装数据库操作

**生成方式：**
```bash
make dao  # 或 gf gen dao
```

**特点：**
- 根据数据库表结构自动生成
- 提供类型安全的 CRUD 接口
- 配合 Model 层的 Entity 和 DO 使用

---

#### 6. Model 层 (internal/model/)
**职责：** 数据模型定义

**子目录说明：**
- `entity/`：数据库表实体（与表结构一一对应）
- `do/`：数据操作对象（用于 DAO 层的查询条件和更新数据）

**生成方式：**
```bash
make dao  # 同时生成 DAO、Entity、DO
```

---

#### 7. 其他重要目录

**packed/ - 资源打包**
- 将 `manifest/` 和 `resource/` 目录的文件打包到二进制
- 通过 `import _ "star/internal/packed"` 自动加载

**consts/ - 常量定义**
- 存放全局常量（状态码、错误码、配置键等）

**utility/ - 工具函数**
- 存放通用的工具函数（加密、验证、转换等）

**hack/ - 开发工具**
- `config.yaml`：gfcli 工具配置（数据库连接、Docker 构建等）
- `hack.mk`：Make 命令定义（构建、生成代码、部署）
- `hack-cli.mk`：CLI 工具自动安装脚本

### 执行流程详解

```
1. 启动阶段
   main.go 
     → connDb()                          # 测试数据库连接
     → cmd.Main.Run()                    # 启动 HTTP Server
       → g.Server()                      # 创建服务器实例
       → group.Middleware()              # 注册中间件
       → group.Bind(hello.NewV1())       # 绑定 Controller

2. 请求处理流程
   HTTP Request (GET /hello)
     → Router 路由匹配                    # 根据 g.Meta 中的 path
     → MiddlewareHandlerResponse         # 统一响应处理中间件
     → Controller.Hello()                # 调用控制器方法
       → Service/Logic 业务逻辑          # （如果有复杂业务）
         → DAO 数据访问                   # （如果需要数据库操作）
           → Database 查询/更新
         ← 返回数据
       ← 返回业务结果
     ← HTTP Response                     # 统一格式的响应
```

**关键点：**
1. 路由自动注册：通过 `g.Meta` 标签 + `group.Bind()` 实现
2. 接口自动绑定：Controller 实现 API 接口，框架自动匹配方法
3. 响应统一处理：`MiddlewareHandlerResponse` 中间件统一封装响应格式

### 关键特性

1. **自动路由绑定**
   - 通过 `g.Meta` 标签定义路由信息
   - 使用 `group.Bind()` 自动注册路由
   - 支持 RESTful 风格和自定义路径

2. **代码自动生成**
   - `make ctrl`：根据 API 定义生成 Controller 骨架
   - `make dao`：根据数据库表生成 DAO/Entity/DO
   - `make service`：根据 Logic 实现生成 Service 接口
   - `make enums`：解析代码生成枚举类型

3. **标准化响应**
   - 使用 `MiddlewareHandlerResponse` 中间件统一处理响应
   - 自动封装成功/失败响应格式
   - 支持自定义响应结构

4. **配置管理**
   - YAML 配置文件（`manifest/config/config.yaml`）
   - 支持多环境配置（开发/测试/生产）
   - 配置热更新（部分配置项）

5. **OpenAPI 支持**
   - 自动生成 API 文档（基于 `g.Meta` 标签）
   - Swagger UI 访问路径：`http://localhost:8000/swagger`
   - API JSON 访问路径：`http://localhost:8000/api.json`

6. **资源打包**
   - 通过 `internal/packed` 将配置和资源文件打包到二进制
   - 支持单文件部署，无需额外配置文件

7. **构建和部署**
   - Makefile 提供丰富的命令（构建/测试/部署）
   - Docker 支持（自动构建镜像）
   - Kubernetes 部署配置（Kustomize）

### 配置说明

**主配置文件：** `manifest/config/config.yaml`

```yaml
server:
  address: ":8000"              # 服务监听地址
  openapiPath: "/api.json"      # OpenAPI JSON 路径
  swaggerPath: "/swagger"       # Swagger UI 路径

logger:
  level: "all"                  # 日志级别（all/debug/info/warn/error）
  stdout: true                  # 是否输出到控制台

database:
  default:
    link: "mysql:root:0307@tcp(127.0.0.1:3306)/go"  # 数据库连接
    debug: true                 # 是否开启 SQL 调试日志

gfcli:
  gen:
    dao:
      - link: "default"         # DAO 生成使用的数据库连接
```

**CLI 工具配置：** `hack/config.yaml`

```yaml
gfcli:
  gen:
    dao:
      - link: "mysql:root:0307@tcp(127.0.0.1:3306)/test"
        descriptionTag: true    # 是否生成字段注释

    build: "-a amd64 -s linux -p temp -ew"  # Docker 构建参数
    tagPrefixes:
      - my.image.pub/my-app     # Docker 镜像标签前缀
```

### 常用命令

**开发命令：**
```bash
# 启动项目
go run main.go

# 生成 Controller（根据 API 定义）
make ctrl

# 生成 DAO/Entity/DO（根据数据库表）
make dao

# 生成 Service 接口（根据 Logic 实现）
make service

# 构建二进制文件
make build

# 更新 GoFrame 到最新版本
make up
```

**部署命令：**
```bash
# 构建 Docker 镜像
make image

# 构建并推送 Docker 镜像
make image.push

# 部署到 Kubernetes
make deploy
```

### 开发流程示例

**添加新的 API 接口：**

1. 定义 API 接口（`api/user/v1/user.go`）
   ```go
   type CreateReq struct {
       g.Meta `path:"/user" method:"post" summary:"创建用户"`
       Name   string `v:"required" json:"name"`
       Email  string `v:"required|email" json:"email"`
   }
   type CreateRes struct {
       Id int64 `json:"id"`
   }
   ```

2. 生成 Controller 骨架
   ```bash
   make ctrl
   ```

3. 实现业务逻辑（`internal/controller/user/user_v1_create.go`）
   ```go
   func (c *ControllerV1) Create(ctx context.Context, req *v1.CreateReq) (res *v1.CreateRes, err error) {
       // 实现创建用户逻辑
       return
   }
   ```

4. 注册路由（`internal/cmd/cmd.go`）
   ```go
   group.Bind(
       hello.NewV1(),
       user.NewV1(),  // 添加新的 Controller
   )
   ```

5. 启动服务并测试
   ```bash
   go run main.go
   # 访问 http://localhost:8000/swagger 查看 API 文档
   ```

### 项目特点总结

这是一个典型的 GoFrame 工程化项目模板，具有以下特点：

✅ 清晰的分层架构（API → Controller → Logic → Service → DAO → Model）  
✅ 代码自动生成（减少重复劳动）  
✅ 接口驱动开发（面向接口编程）  
✅ 配置化管理（YAML 配置文件）  
✅ 标准化响应（统一的响应格式）  
✅ 完善的工具链（Makefile + gfcli）  
✅ 容器化支持（Docker + Kubernetes）  
✅ API 文档自动生成（OpenAPI + Swagger）  

适合用于构建中大型的企业级 Web 应用和微服务项目。
