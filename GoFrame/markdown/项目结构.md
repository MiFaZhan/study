
## GoFrame 项目架构概览

这是一个基于 GoFrame v2.7.1 的标准单体应用（SingleRepo）项目，采用了经典的分层架构设计。

本项目是一个单词管理系统，包含用户注册/登录、JWT 鉴权、单词 CRUD 等完整功能。

### 分层架构详解

#### 1. API 层 (api/)
**职责：** 定义对外的 API 接口契约

**实际业务模块：**

**users 模块（用户管理）：**
```go
// 注册接口
type RegisterReq struct {
    g.Meta   `path:"users/register" method:"post" sm:"注册" tags:"用户"`
    Username string `json:"username" v:"required|length:3,12" dc:"用户名"`
    Password string `json:"password" v:"required|length:6,16" dc:"密码"`
    Email    string `json:"email" v:"required|email" dc:"邮箱"`
}

// 登录接口
type LoginReq struct {
    g.Meta   `path:"users/login" method:"post" sm:"登录" tags:"用户"`
    Username string `json:"username" v:"required|length:3,12"`
    Password string `json:"password" v:"required|length:6,16"`
}
type LoginRes struct {
    Token string `json:"token" dc:"在需要鉴权的接口中header加入Authorization: token"`
}
```

**words 模块（单词管理）：**
```go
// 创建单词
type CreateReq struct {
    g.Meta             `path:"words" method:"post" sm:"创建" tags:"单词"`
    Word               string           `json:"word" v:"required|length:1,100"`
    Definition         string           `json:"definition" v:"required|length:1,300"`
    ExampleSentence    string           `json:"example_sentence" v:"required|length:1,300"`
    ChineseTranslation string           `json:"chinese_translation" v:"required|length:1,300"`
    Pronunciation      string           `json:"pronunciation" v:"required|length:1,100"`
    ProficiencyLevel   ProficiencyLevel `json:"proficiency_level" v:"required|between:1,5"`
}

// 单词列表（支持分页和模糊查询）
type ListReq struct {
    g.Meta `path:"words" method:"get" sm:"列表" tags:"单词"`
    Word   string `json:"word" v:"length:1,100" dc:"模糊查询单词"`
    Page   int    `json:"page" v:"min:1" dc:"页码，默认1"`
    Size   int    `json:"size" v:"between:1,100" dc:"每页数量，默认10"`
}
```

**account 模块（账户信息）：**
```go
// 获取当前用户信息（需要鉴权）
type InfoReq struct {
    g.Meta `path:"account/info" method:"get" sm:"获取信息" tags:"用户"`
}
type InfoRes struct {
    Username  string      `json:"username" dc:"用户名"`
    Email     string      `json:"email" dc:"邮箱"`
    CreatedAt *gtime.Time `json:"created" dc:"创建时间"`
    UpdatedAt *gtime.Time `json:"update" dc:"更新时间"`
}
```

**特点：**
- 使用 `g.Meta` 标签定义路由、HTTP 方法、文档说明
- 使用 `v` 标签定义验证规则（required、length、email、between 等）
- 使用 `dc` 标签定义字段描述（用于生成 API 文档）
- 支持版本管理（v1, v2, v3...）
- 通过 `gf gen ctrl` 命令自动生成接口定义和 Controller 骨架

---

#### 2. Controller 层 (internal/controller/)
**职责：** 接收 HTTP 请求，调用业务逻辑，返回响应

**文件组织模式：**
- `{module}.go`：手动编写的业务代码（可自由修改）
- `{module}_new.go`：自动生成的构造函数（不可修改）
- `{module}_v1_{method}.go`：自动生成的接口实现骨架（可修改）

**实际示例（users 控制器）：**
```go
// users_new.go（自动生成）
func NewV1() users.IUsersV1 {
    return &ControllerV1{}
}

// users_v1_login.go（可修改）
func (c *ControllerV1) Login(ctx context.Context, req *v1.LoginReq) (res *v1.LoginRes, err error) {
    res, err = users.New().Login(ctx, req)  // 调用 Logic 层
    return
}
```

**特点：**
- 实现 API 层定义的接口
- 通过 `cmd.go` 中的 `group.Bind()` 自动注册路由
- 使用 `MiddlewareHandlerResponse` 中间件统一处理响应格式
- Controller 层通常只做参数传递，具体业务逻辑在 Logic 层实现

---

#### 3. Logic 层 (internal/logic/)
**职责：** 实现复杂的业务逻辑

**实际业务实现：**

**users 业务逻辑：**
- `users.go`：业务对象定义 + 密码加密方法
  ```go
  func (u *Users) encryptPassword(password string) string {
      return gmd5.MustEncryptString(password)
  }
  ```
- `users_register.go`：注册逻辑（密码加密 + 入库）
- `users_login.go`：登录逻辑（验证密码 + 生成 JWT Token）
- `users_account.go`：获取账户信息逻辑

**words 业务逻辑：**
- `words_create.go`：创建单词逻辑（关联用户 ID）
- `words_list.go`：单词列表逻辑（分页 + 模糊查询）
- `words_detail.go`：单词详情逻辑
- `words_update.go`：更新单词逻辑
- `words_delete.go`：删除单词逻辑

**middleware 中间件：**
- `auth.go`：JWT 鉴权中间件
  ```go
  func Auth(r *ghttp.Request) {
      var tokenString = r.Header.Get("Authorization")
      token, err := jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
          return []byte(consts.JwtKey), nil
      })
      if err != nil || !token.Valid {
          r.Response.WriteStatus(http.StatusForbidden)
          r.Exit()
      }
      r.Middleware.Next()
  }
  ```

**特点：**
- 实现 Service 层定义的接口（如果有）
- 可被多个 Controller 复用
- 不直接处理 HTTP 请求，专注于业务逻辑
- 调用 DAO 层进行数据库操作
- 包含业务规则、事务处理、数据组装等

---

#### 4. Service 层 (internal/service/)
**职责：** 定义业务服务的接口（interface）

**当前状态：** 本项目暂未使用 Service 层，Controller 直接调用 Logic

**设计原则：**
- 只包含方法签名，不包含具体实现
- 遵循"面向接口编程"原则
- Controller 依赖 Service 接口，而非直接依赖 Logic 实现
- 便于单元测试和依赖注入

**生成方式：**
```bash
make service  # 或 gf gen service
```

---

#### 5. DAO 层 (internal/dao/)
**职责：** 数据访问对象，封装数据库操作

**实际生成的 DAO：**
- `users.go`：Users 表的 DAO 对外接口
- `words.go`：Words 表的 DAO 对外接口
- `internal/users.go`：Users 表的 DAO 内部实现（自动生成）
- `internal/words.go`：Words 表的 DAO 内部实现（自动生成）

**使用示例：**
```go
// 查询用户
user, err := dao.Users.Ctx(ctx).Where("username", username).One()

// 创建单词
_, err := dao.Words.Ctx(ctx).Data(do.Words{
    Word:       req.Word,
    Definition: req.Definition,
    // ...
}).Insert()

// 分页查询
list, err := dao.Words.Ctx(ctx).
    Where("uid", uid).
    WhereLike("word", "%"+req.Word+"%").
    Page(req.Page, req.Size).
    All()
```

**生成方式：**
```bash
make dao  # 或 gf gen dao
```

**特点：**
- 根据数据库表结构自动生成
- 提供类型安全的 CRUD 接口
- 配合 Model 层的 Entity 和 DO 使用
- 支持链式调用、条件构建、分页查询等

### 执行流程详解

```
1. 启动阶段
   main.go 
     → connDb()                          # 测试数据库连接
     → cmd.Main.Run()                    # 启动 HTTP Server
       → g.Server()                      # 创建服务器实例
       → group.Middleware()              # 注册全局中间件
         - MiddlewareHandlerResponse     # 统一响应处理
       → group.Bind()                    # 绑定公开接口（无需鉴权）
         - users.NewV1()                 # 注册/登录接口
       → group.Middleware(Auth)          # 注册鉴权中间件
       → group.Bind()                    # 绑定需要鉴权的接口
         - account.NewV1()               # 账户信息接口
         - words.NewV1()                 # 单词管理接口

2. 请求处理流程（无需鉴权）
   HTTP Request (POST /v1/users/register)
     → Router 路由匹配                    # 根据 g.Meta 中的 path
     → MiddlewareHandlerResponse         # 统一响应处理中间件
     → Controller.Register()             # 调用控制器方法
       → Logic.Register()                # 调用业务逻辑
         → encryptPassword()             # 密码加密
         → DAO.Insert()                  # 插入数据库
           → Database 执行 INSERT
         ← 返回插入结果
       ← 返回业务结果
     ← HTTP Response                     # 统一格式的响应

3. 请求处理流程（需要鉴权）
   HTTP Request (GET /v1/words?page=1&size=10)
     → Router 路由匹配
     → MiddlewareHandlerResponse         # 统一响应处理中间件
     → Middleware.Auth()                 # JWT 鉴权中间件
       → 解析 Authorization Header
       → 验证 JWT Token
       → 提取用户信息
     → Controller.List()                 # 调用控制器方法
       → Logic.List()                    # 调用业务逻辑
         → DAO.Page().All()              # 分页查询数据库
           → Database 执行 SELECT
         ← 返回查询结果
       ← 返回业务结果
     ← HTTP Response                     # 统一格式的响应

4. 路由分组结构
   /v1
     ├── /users/register (POST)          # 公开接口
     ├── /users/login (POST)             # 公开接口
     └── [Auth 中间件]                    # 以下接口需要鉴权
         ├── /account/info (GET)         # 获取用户信息
         ├── /words (POST)               # 创建单词
         ├── /words (GET)                # 单词列表
         ├── /words/{id} (GET)           # 单词详情
         ├── /words/{id} (PUT)           # 更新单词
         └── /words/{id} (DELETE)        # 删除单词
```

**关键点：**
1. 路由自动注册：通过 `g.Meta` 标签 + `group.Bind()` 实现
2. 接口自动绑定：Controller 实现 API 接口，框架自动匹配方法
3. 响应统一处理：`MiddlewareHandlerResponse` 中间件统一封装响应格式
4. 分组鉴权：通过路由分组实现部分接口需要鉴权，部分接口公开访问
5. JWT 鉴权：使用 JWT Token 进行用户身份验证

### 关键特性

1. **自动路由绑定**
   - 通过 `g.Meta` 标签定义路由信息
   - 使用 `group.Bind()` 自动注册路由
   - 支持 RESTful 风格和自定义路径

2. **代码自动生成**
   - `make ctrl`：根据 API 定义生成 Controller 骨架
   - `make dao`：根据数据库表生成 DAO/Entity/DO
   - `make service`：根据 Logic 实现生成 Service 接口
   - `make enums`：解析代码生成枚举类型

3. **标准化响应**
   - 使用 `MiddlewareHandlerResponse` 中间件统一处理响应
   - 自动封装成功/失败响应格式
   - 支持自定义响应结构

4. **配置管理**
   - YAML 配置文件（`manifest/config/config.yaml`）
   - 支持多环境配置（开发/测试/生产）
   - 配置热更新（部分配置项）

5. **OpenAPI 支持**
   - 自动生成 API 文档（基于 `g.Meta` 标签）
   - Swagger UI 访问路径：`http://localhost:8000/swagger`
   - API JSON 访问路径：`http://localhost:8000/api.json`

6. **资源打包**
   - 通过 `internal/packed` 将配置和资源文件打包到二进制
   - 支持单文件部署，无需额外配置文件

7. **构建和部署**
   - Makefile 提供丰富的命令（构建/测试/部署）
   - Docker 支持（自动构建镜像）
   - Kubernetes 部署配置（Kustomize）

### 开发流程示例

**添加新的 API 接口：**

1. 定义 API 接口（`api/user/v1/user.go`）
   ```go
   type CreateReq struct {
       g.Meta `path:"/user" method:"post" summary:"创建用户" tags:"用户"`
       Name   string `v:"required|length:3,12" json:"name" dc:"用户名"`
       Email  string `v:"required|email" json:"email" dc:"邮箱"`
   }
   type CreateRes struct {
       Id int64 `json:"id" dc:"用户ID"`
   }
   ```

2. 生成 Controller 骨架
   ```bash
   make ctrl
   # 自动生成：
   # - api/user/user.go（接口定义）
   # - internal/controller/user/user_new.go（构造函数）
   # - internal/controller/user/user_v1_create.go（实现骨架）
   ```

3. 实现业务逻辑（`internal/logic/user/user_create.go`）
   ```go
   func (u *User) Create(ctx context.Context, req *v1.CreateReq) (res *v1.CreateRes, err error) {
       // 实现创建用户逻辑
       result, err := dao.Users.Ctx(ctx).Data(do.Users{
           Name:  req.Name,
           Email: req.Email,
       }).Insert()
       
       if err != nil {
           return nil, err
       }
       
       id, _ := result.LastInsertId()
       return &v1.CreateRes{Id: id}, nil
   }
   ```

4. Controller 调用 Logic（`internal/controller/user/user_v1_create.go`）
   ```go
   func (c *ControllerV1) Create(ctx context.Context, req *v1.CreateReq) (res *v1.CreateRes, err error) {
       res, err = user.New().Create(ctx, req)
       return
   }
   ```

5. 注册路由（`internal/cmd/cmd.go`）
   ```go
   group.Bind(
       users.NewV1(),
       user.NewV1(),  // 添加新的 Controller
   )
   ```

6. 启动服务并测试
   ```bash
   go run main.go
   # 访问 http://localhost:8000/swagger 查看 API 文档
   # 或导入 apifox-openapi.json 到 Apifox 进行测试
   ```

### 实际业务功能说明

本项目实现了一个完整的单词管理系统，包含以下功能：

**1. 用户管理**
- 用户注册（密码 MD5 加密）
- 用户登录（生成 JWT Token）
- 获取用户信息（需要鉴权）

**2. 单词管理（需要鉴权）**
- 创建单词（关联当前用户）
- 更新单词
- 删除单词
- 单词列表（支持分页和模糊查询）
- 单词详情

**3. 技术特性**
- JWT 身份验证
- 密码加密存储（MD5）
- 参数验证（长度、格式、范围等）
- 国际化错误提示（中文）
- RESTful API 设计
- OpenAPI 文档自动生成

**4. 数据库表结构**

**users 表：**
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL,
    created_at DATETIME,
    updated_at DATETIME
);
```

**words 表：**
```sql
CREATE TABLE words (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL,                    -- 用户ID
    word VARCHAR(100) NOT NULL,
    definition VARCHAR(300) NOT NULL,
    example_sentence VARCHAR(300) NOT NULL,
    chinese_translation VARCHAR(300) NOT NULL,
    pronunciation VARCHAR(100) NOT NULL,
    proficiency_level TINYINT NOT NULL,  -- 熟练度 1-5
    created_at DATETIME,
    updated_at DATETIME
);
```
