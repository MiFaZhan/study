


          
## 为什么是 `document.xxx`？

这是因为 **`document` 是整个网页的入口点**。

### 什么是 `document`？

`document` 是一个 **全局对象**，它代表了 **整个 HTML 文档**（也就是整个网页）。

```
浏览器窗口
┌─────────────────────────────────────┐
│           window 对象                │
│  ┌─────────────────────────────────┐│
│  │      document 对象               ││  ← 整个网页的"根"
│  │  ┌───────────────────────────┐  ││
│  │  │  <html>                   │  ││
│  │  │    <head>...</head>       │  ││
│  │  │    <body>                 │  ││
│  │  │      <div id="app"></div> │  ││  ← 文档中的元素
│  │  │    </body>                │  ││
│  │  │  </html>                  │  ││
│  │  └───────────────────────────┘  ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 为什么从 `document` 开始？

因为 **DOM 是树形结构**：

```
document (根节点)
  └─ <html>
       ├─ <head>
       │    └─ <title>
       └─ <body>
            ├─ <div id="app">
            ├─ <header>
            └─ <footer>
```

要访问树上的任何元素，必须从 **根节点** 开始，就像：
- 要找公司里的某个人 → 先找到公司（document）
- 然后逐层往下找部门、员工

### 关系图

```
window.document
     │
     ├── .getElementById()      → 通过 ID 查找
     ├── .getElementsByClassName() → 通过类名查找
     ├── .querySelector()       → CSS 选择器查找
     ├── .createElement()       → 创建新元素
     └── .body                  → 获取 <body> 元素
```

### 小结

| 对象 | 含义 |
|------|------|
| `window` | 浏览器窗口（最顶层） |
| `document` | 当前网页文档 |
| `element` | 网页中的具体标签 |

所以 `document.getElementById()` 的完整理解是：
> **"在当前网页文档中，通过 ID 查找一个元素"**
