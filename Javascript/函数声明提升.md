


          
## 什么是"提升"（Hoisting）

**提升**是 JavaScript 的一种执行机制：代码在正式执行前，引擎会先"读取"声明，将它们放到作用域顶部。

---

## 函数声明的提升

```javascript
// ✅ 可以正常调用 — 因为函数声明会被提升
console.log(add(2, 3)); // 5

function add(a, b) { 
    return a + b; 
}
```

**实际执行顺序（引擎处理的顺序）：**
```javascript
function add(a, b) { return a + b; } // 提升到顶部

console.log(add(2, 3)); // 5 — 调用时函数已经存在
```

---

## 函数表达式的提升

```javascript
// ❌ 报错 — TypeError: multiply is not a function
console.log(multiply(3, 4));

const multiply = function(a, b) { 
    return a * b; 
};
```

**实际执行顺序（引擎处理的顺序）：**
```javascript
const multiply; // 只提升变量声明，不提升赋值

console.log(multiply(3, 4)); // multiply 是 undefined，不是函数！

multiply = function(a, b) { return a * b; }; // 赋值在调用之后
```

---

## 对比总结

| 代码形式 | 会被提升吗 | 能在定义前调用吗 |
|----------|------------|------------------|
| `function add() {}` | ✅ 整个函数 | ✅ 可以 |
| `const add = function() {}` | ❌ 只有变量 | ❌ 不可以 |
| `const add = () => {}` | ❌ 只有变量 | ❌ 不可以 |

---

## 实际开发建议

虽然函数声明可以提升，但**不建议依赖这个特性**，因为：
1. 代码可读性差（调用在定义之前）
2. 容易产生意外错误

**最佳实践**：先定义，后调用，保持代码顺序清晰。
